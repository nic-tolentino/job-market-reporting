## All Changes Made

### Architecture Refinements (Gemini Feedback)

1. **Dynamic Chunk Sizing** (Section 2.3, 2.5)
   - Added chunk size strategy table: Apify (5,000), ATS (500), Default (2,000)
   - Updated `createBronzeManifest()` with source-based dynamic chunking
   - Removed hardcoded `chunkSize = 500` from `BronzeGcsRepository`

2. **Parquet Recommendation** (Section 2.3, 9.1)
   - Added NDJSON vs Parquet comparison table
   - Expanded Section 9.1 with migration path and dependencies
   - Added decision criteria for when to migrate

3. **Metadata Schema Enhancements** (Section 2.3, 2.4)
   - Added `uncompressedSizeBytes` and `compressedSizeBytes` fields
   - Added `compressionRatio` field (FLOAT64)
   - Added `ProcessingStatus` enum (PENDING, COMPLETED, FAILED)
   - Updated BigQuery schema with clustering on `source` and `ingested_at`

4. **Orphaned File Handling** (Section 2.5)
   - Wrapped `saveIngestion()` in try-catch with cleanup logic
   - Added `deleteFromGcs()` helper method
   - Ensures GCS/BigQuery consistency on failures

5. **BigQuery-Based Reprocessing** (Section 2.5)
   - Added `reprocessHistoricalDataBigQuery()` method
   - Documented two approaches: PARSE_JSON vs schema-based external table
   - Recommended schema-based approach for production (>100k records)

6. **Processing Status Tracking** (New Section 9.4)
   - Added guidance on leveraging `processingStatus` field
   - Enables retry logic and pipeline health monitoring

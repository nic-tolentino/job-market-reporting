**Status:** Proposed

## Problem Statement

During Phase 2.5 (Type-Safe Query Results), we introduced contract tests to catch mismatches between queries and mappers. However, a production bug slipped through:

**Bug:** `JobMapper.mapJobDetails()` tried to read the `source` field, but `JobQueries.getDetailsSql()` didn't include it in the SELECT clause.

**Impact:** Production error: `IllegalArgumentException: Field with name 'source' was not found`

**Root Cause:** The contract test had a **fatal assumption**:

```kotlin
// ❌ WRONG: Assumed j.* always works
// Note: getDetailsSql uses "j.*" which selects all job fields automatically
// We only need to verify the company fields (which use aliases) are present
```

This assumption was wrong because:
1. `j.*` with JOINs doesn't guarantee all fields in BigQuery
2. The test only verified company alias fields, not job fields
3. Manual `requiredFields` lists can become stale

**Lesson:** Contract tests are only as good as their field lists. Manual maintenance is error-prone.

---

## Solution Overview

**Automatically extract field requirements from mapper source code** instead of manually maintaining field lists.

### Current State (Manual - Error-Prone)

```kotlin
// JobQueries.kt - Manual field list (can become stale)
fun getDetailsSql(...): JobDetailsQuery {
    return JobDetailsQuery(
        sql = "SELECT ...",
        requiredFields = listOf(  // ← Humans forget to update this
            JobFields.JOB_ID,
            JobFields.TITLE,
            // ... forgot SOURCE!
        )
    )
}

// JobQueryMapperContractTest.kt - Manual verification
@Test
fun `getDetailsSql includes all required fields`() {
    val query = JobQueries.getDetailsSql(...)
    val requiredFields = listOf(  // ← Another manual list!
        JobFields.JOB_ID,
        JobFields.TITLE,
        // ... also forgot SOURCE!
    )
    requiredFields.forEach { field ->
        assertTrue(query.sql.contains(field))
    }
}
```

**Problem:** Two manual lists that can diverge from what the mapper actually reads.

### Target State (Auto-Generated - Single Source of Truth)

```kotlin
// JobQueryMapperContractTest.kt - Auto-extract from mapper source
@Test
fun `getDetailsSql includes all fields JobMapper reads`() {
    // Parse JobMapper.kt source to find all JobFields.X accesses
    val mapperSource = readSourceFile("JobMapper.kt")
    val fieldsMapperReads = extractFieldReferences(mapperSource, "JobFields")
    
    // Get the query's requiredFields list
    val query = JobQueries.getDetailsSql("db", "jobs", "companies")
    val fieldsQueryProvides = query.requiredFields
    
    // Every field the mapper reads must be provided by the query
    val missingFields = fieldsMapperReads - fieldsQueryProvides.toSet()
    
    assertTrue(
        missingFields.isEmpty(),
        "JobMapper reads fields not in JobQueries.requiredFields: $missingFields"
    )
}
```

**Benefit:** Single source of truth (the mapper code itself). Can't forget to update.

## Coding Standards

### ⚠️ Critical: No Static Imports in Mappers

**Rule:** Mappers must NOT use static imports for field objects.

```kotlin
// ✅ GOOD - FieldExtractor can parse this
import com.techmarket.persistence.JobFields
import com.techmarket.persistence.CompanyAliases

val source = r.get(JobFields.SOURCE).stringValue
val name = r.get(CompanyAliases.NAME).stringValue

// ❌ BAD - FieldExtractor CANNOT parse this
import com.techmarket.persistence.JobFields.SOURCE
import com.techmarket.persistence.CompanyAliases.NAME

val source = r.get(SOURCE).stringValue  // Can't tell which object this came from!
val name = r.get(NAME).stringValue
```

**Why:** The FieldExtractor regex `JobFields\.([A-Z_]+)` requires the object prefix to identify which fields are being accessed.

**Enforcement:** Add to PR checklist and code review guidelines.

### Step 1: Create FieldExtractor Utility (1 hour)

**File:** `backend/src/test/kotlin/com/techmarket/contract/FieldExtractor.kt`

```kotlin
package com.techmarket.contract

import java.io.File

/**
 * Extracts field references from Kotlin source code.
 * Used for automated contract testing between queries and mappers.
 * 
 * Supports extracting from:
 * - JobFields.FIELD_NAME
 * - CompanyFields.FIELD_NAME
 * - CompanyAliases.FIELD_NAME
 * - SalaryFields.FIELD_NAME
 */
object FieldExtractor {

    /**
     * Extracts all references to field objects from Kotlin source code.
     *
     * @param sourceCode The Kotlin source code to parse
     * @param fieldObjectNames The object names (e.g., "JobFields", "CompanyFields", "CompanyAliases")
     * @return Set of field names (e.g., ["SOURCE", "TITLE", "JOB_ID"])
     */
    fun extractFields(sourceCode: String, vararg fieldObjectNames: String): Set<String> {
        // Build regex that matches any of the field objects: (JobFields|CompanyFields|CompanyAliases).FIELD_NAME
        val objectPattern = fieldObjectNames.joinToString("|", "(", ")")
        val regex = Regex("""$objectPattern\.([A-Z_]+)""")
        
        return regex.findAll(sourceCode)
            .map { it.groupValues[1] }
            .toSet()
    }

    /**
     * Reads a source file and extracts field references.
     *
     * @param relativePath Path relative to project root (e.g., "src/main/kotlin/.../JobMapper.kt")
     * @param fieldObjectName The object name (e.g., "JobFields")
     * @return Set of field names
     */
    fun extractFieldsFromFile(relativePath: String, fieldObjectName: String): Set<String> {
        val projectRoot = System.getProperty("user.dir")
            ?: throw IllegalStateException("Cannot determine project root")
        
        val filePath = listOf(
            File("$projectRoot/$relativePath"),
            File("$projectRoot/backend/$relativePath")
        ).find { it.exists() }
            ?: throw IllegalStateException("Source file not found: $relativePath")
        
        val sourceCode = filePath.readText()
        return extractFields(sourceCode, fieldObjectName)
    }

    /**
     * Extracts fields accessed by a specific method in a source file.
     * More granular than extractFieldsFromFile.
     *
     * @param relativePath Path to the source file
     * @param fieldObjectName The object name (e.g., "JobFields")
     * @param methodName The method name to search within (e.g., "mapJobDetails")
     * @return Set of field names accessed within that method
     */
    fun extractFieldsFromMethod(
        relativePath: String,
        fieldObjectName: String,
        methodName: String
    ): Set<String> {
        val projectRoot = System.getProperty("user.dir")
            ?: throw IllegalStateException("Cannot determine project root")
        
        val filePath = listOf(
            File("$projectRoot/$relativePath"),
            File("$projectRoot/backend/$relativePath")
        ).find { it.exists() }
            ?: throw IllegalStateException("Source file not found: $relativePath")
        
        val sourceCode = filePath.readText()
        
        // Find the method body (simple heuristic: from "fun methodName" to next "fun" or end of class)
        val methodRegex = Regex("""fun\s+$methodName\([^)]*\)[^:]*:[^ {]*\s*\{([\s\S]*?)\n    \}""")
        val methodMatch = methodRegex.find(sourceCode)
            ?: throw IllegalStateException("Method not found: $methodName")
        
        val methodBody = methodMatch.groupValues[1]
        return extractFields(methodBody, fieldObjectName)
    }
}
```

**Design Decisions:**
- Uses regex parsing (simple, no dependencies)
- Works with source files (no compilation needed)
- Supports both full-file and method-level extraction
- Returns `Set<String>` for easy set operations

package com.techmarket.persistence.job

import com.techmarket.contract.FieldExtractor
import com.techmarket.persistence.CompanyFields
import com.techmarket.persistence.JobFields
import org.junit.jupiter.api.Test
import org.junit.jupiter.api.Assertions.assertTrue

/**
 * Automated contract tests to ensure JobQueries and JobMapper are in sync.
 * 
 * Uses source code parsing to automatically detect which fields the mapper reads,
 * eliminating manual field list maintenance.
 */
class JobQueryMapperContractTest {

    @Test
    fun `getDetailsSql includes all fields JobMapper.mapJobDetails reads`() {
        // Auto-extract fields that JobMapper.mapJobDetails reads from JobFields
        val fieldsMapperReads = FieldExtractor.extractFieldsFromMethod(
            relativePath = "src/main/kotlin/com/techmarket/persistence/job/JobMapper.kt",
            fieldObjectName = "JobFields",
            methodName = "mapJobDetails"
        )

        // Get the query's requiredFields list
        val query = JobQueries.getDetailsSql("dataset", "jobs", "companies")
        val fieldsQueryProvides = query.requiredFields.toSet()

        // Every field the mapper reads must be provided by the query
        val missingFields = fieldsMapperReads - fieldsQueryProvides

        assertTrue(
            missingFields.isEmpty(),
            "JobMapper.mapJobDetails reads fields not in JobQueries.getDetailsSql.requiredFields: $missingFields\n" +
            "Fix: Add these fields to the requiredFields list in JobQueries.getDetailsSql()"
        )

        // Also verify company field aliases (from the JOIN)
        val requiredCompanyFields = mapOf(
            "comp_name" to CompanyFields.NAME,
            "comp_logo" to CompanyFields.LOGO_URL,
            "comp_desc" to CompanyFields.DESCRIPTION,
            "comp_web" to CompanyFields.WEBSITE,
            "comp_hiringLocations" to CompanyFields.HIRING_LOCATIONS,
            "comp_hqCountry" to CompanyFields.HQ_COUNTRY,
            "comp_verificationLevel" to CompanyFields.VERIFICATION_LEVEL
        )

        val sql = query.sql
        requiredCompanyFields.forEach { (alias, field) ->
            assertTrue(
                sql.contains(alias, ignoreCase = true),
                "Company field alias '$alias' (for $field) is accessed by JobMapper but not found in getDetailsSql SELECT clause."
            )
        }
    }

    @Test
    fun `getSimilarSql includes all fields JobMapper reads for similar roles`() {
        // Auto-extract fields from the similar roles mapping code
        val fieldsMapperReads = FieldExtractor.extractFieldsFromMethod(
            relativePath = "src/main/kotlin/com/techmarket/persistence/job/JobMapper.kt",
            fieldObjectName = "JobFields",
            methodName = "mapJobDetails"  // Includes similarRoles mapping
        )

        // Test both branches (with and without tech list)
        val queryWithoutTech = JobQueries.getSimilarSql("dataset", "jobs", emptyList())
        val queryWithTech = JobQueries.getSimilarSql("dataset", "jobs", listOf("kotlin"))

        val fieldsQueryProvides = queryWithoutTech.requiredFields.toSet()

        val missingFields = fieldsMapperReads - fieldsQueryProvides

        assertTrue(
            missingFields.isEmpty(),
            "JobMapper reads fields not in JobQueries.getSimilarSql.requiredFields: $missingFields"
        )
    }
}

**Key Changes:**
- ✅ Auto-extracts fields from mapper source

### Step 5: Address Known Limitations (30 min)

#### 3. Regex Fragility

The `extractFieldsFromMethod` regex uses a simple "scan-between" approach instead of strict indentation matching:

```kotlin
fun extractFieldsFromMethod(
    relativePath: String,
    fieldObjectName: String,
    methodName: String
): Set<String> {
    val sourceCode = filePath.readText()
    
    // Find method start
    val methodStartRegex = Regex("""fun\s+$methodName\s*\(""")
    val startMatch = methodStartRegex.find(sourceCode)
        ?: throw IllegalStateException("Method not found: $methodName")
    
    // Find method end (next 'fun' at start of line or end of file)
    val remainingCode = sourceCode.substring(startMatch.range.last)
    val methodEndRegex = Regex("""\n    fun |\n\}$""")
    val endMatch = methodEndRegex.find(remainingCode)
    
    val methodBody = if (endMatch != null) {
        remainingCode.substring(0, endMatch.range.first)
    } else {
        remainingCode
    }
    
    return extractFields(methodBody, *fieldObjectNames)
}
```

**Why:** More resilient to formatting changes than indentation-based regex.

#### 4. SalaryMapper Deep-Dive

Since `JobMapper` delegates to `SalaryMapper`, the FieldExtractor needs to understand this relationship:

```kotlin
// In FieldExtractor.kt
object FieldExtractor {
    
    // Salary fields that are implicitly required when getSalaryOrNull is called
    private val SALARY_FIELDS = setOf("AMOUNT", "CURRENCY", "PERIOD", "SOURCE", "IS_GROSS")
    
    fun extractFieldsWithDependencies(
        sourceCode: String, 
        vararg fieldObjectNames: String
    ): Set<String> {
        val explicitFields = extractFields(sourceCode, *fieldObjectNames)
        
        // If mapper calls getSalaryOrNull, it implicitly requires all salary fields
        val allFields = explicitFields.toMutableSet()
        if (sourceCode.contains("getSalaryOrNull")) {
            allFields.addAll(SALARY_FIELDS)
        }
        
        return allFields
    }
}
```

**Why:** Ensures contract tests verify that queries include all salary STRUCT fields when salary extraction is used.

### 2. Stale Field Detection

Add a reverse check to detect when queries select fields that no mapper reads:

```kotlin
@Test
fun `getDetailsSql does not select stale fields`() {
    val fieldsMapperReads = FieldExtractor.extractFieldsFromMethod(
        "src/main/kotlin/.../JobMapper.kt",
        "mapJobDetails",
        "JobFields", "CompanyAliases"
    )
    
    val query = JobQueries.getDetailsSql("db", "jobs", "companies")
    val fieldsQuerySelects = query.requiredFields.toSet()
    
    // Warn about fields that are selected but never read
    val staleFields = fieldsQuerySelects - fieldsMapperReads
    
    if (staleFields.isNotEmpty()) {
        println("⚠️  WARNING: getDetailsSql selects fields that JobMapper doesn't read: $staleFields")
        println("   Consider removing these to reduce BigQuery costs")
        // Don't fail the test - this is an optimization, not a bug
    }
}
```

**Benefit:** Keeps queries lean and reduces BigQuery scanning costs.

```kotlin
fun <T> executeQuery(query: QueryWithRequiredFields, mapper: (Row) -> T): List<T> {
    val result = bigQuery.query(query.sql)
    
    // Validate all required fields are present in result schema
    query.requiredFields.forEach { field ->
        if (result.schema.fields.none { it.name == field }) {
            throw QuerySchemaMismatchException(query, field)
        }
    }
    
    return result.iterateAll().map { mapper(it) }
}
```

**Benefit:** Fails fast in staging, not silently in production.

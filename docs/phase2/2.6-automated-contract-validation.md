## Problem Statement

During Phase 2.5 (Type-Safe Query Results), we introduced contract tests to catch mismatches between queries and mappers. However, a production bug slipped through:

**Bug:** `JobMapper.mapJobDetails()` tried to read the `source` field, but `JobQueries.getDetailsSql()` didn't include it in the SELECT clause.

**Impact:** Production error: `IllegalArgumentException: Field with name 'source' was not found`

**Root Cause:** The contract test had a **fatal assumption**:

```kotlin
// ❌ WRONG: Assumed j.* always works
// Note: getDetailsSql uses "j.*" which selects all job fields automatically
// We only need to verify the company fields (which use aliases) are present
```

This assumption was wrong because:
1. `j.*` with JOINs doesn't guarantee all fields in BigQuery
2. The test only verified company alias fields, not job fields
3. Manual `requiredFields` lists can become stale

**Lesson:** Contract tests are only as good as their field lists. Manual maintenance is error-prone.

---

## Solution Overview

**Automatically extract field requirements from mapper source code** instead of manually maintaining field lists.

### Current State (Manual - Error-Prone)

```kotlin
// JobQueries.kt - Manual field list (can become stale)
fun getDetailsSql(...): JobDetailsQuery {
    return JobDetailsQuery(
        sql = "SELECT ...",
        requiredFields = listOf(  // ← Humans forget to update this
            JobFields.JOB_ID,
            JobFields.TITLE,
            // ... forgot SOURCE!
        )
    )
}

// JobQueryMapperContractTest.kt - Manual verification
@Test
fun `getDetailsSql includes all required fields`() {
    val query = JobQueries.getDetailsSql(...)
    val requiredFields = listOf(  // ← Another manual list!
        JobFields.JOB_ID,
        JobFields.TITLE,
        // ... also forgot SOURCE!
    )
    requiredFields.forEach { field ->
        assertTrue(query.sql.contains(field))
    }
}
```

**Problem:** Two manual lists that can diverge from what the mapper actually reads.

### Target State (Auto-Generated - Single Source of Truth)

```kotlin
// JobQueryMapperContractTest.kt - Auto-extract from mapper source
@Test
fun `getDetailsSql includes all fields JobMapper reads`() {
    // Parse JobMapper.kt source to find all JobFields.X accesses
    val mapperSource = readSourceFile("JobMapper.kt")
    val fieldsMapperReads = extractFieldReferences(mapperSource, "JobFields")
    
    // Get the query's requiredFields list
    val query = JobQueries.getDetailsSql("db", "jobs", "companies")
    val fieldsQueryProvides = query.requiredFields
    
    // Every field the mapper reads must be provided by the query
    val missingFields = fieldsMapperReads - fieldsQueryProvides.toSet()
    
    assertTrue(
        missingFields.isEmpty(),
        "JobMapper reads fields not in JobQueries.requiredFields: $missingFields"
    )
}
```

**Benefit:** Single source of truth (the mapper code itself). Can't forget to update.

## Coding Standards

### ⚠️ Critical: No Static Imports in Mappers

**Rule:** Mappers must NOT use static imports for field objects.

```kotlin
// ✅ GOOD - FieldExtractor can parse this
import com.techmarket.persistence.JobFields
import com.techmarket.persistence.CompanyAliases

val source = r.get(JobFields.SOURCE).stringValue
val name = r.get(CompanyAliases.NAME).stringValue

// ❌ BAD - FieldExtractor CANNOT parse this
import com.techmarket.persistence.JobFields.SOURCE
import com.techmarket.persistence.CompanyAliases.NAME

val source = r.get(SOURCE).stringValue  // Can't tell which object this came from!
val name = r.get(NAME).stringValue
```

**Why:** The FieldExtractor regex `JobFields\.([A-Z_]+)` requires the object prefix to identify which fields are being accessed.

**Enforcement:** Add to PR checklist and code review guidelines.

### Step 1: Create FieldExtractor Utility (1 hour)

**File:** `backend/src/test/kotlin/com/techmarket/contract/FieldExtractor.kt`

```kotlin
package com.techmarket.contract

/**
 * Utility for extracting field references from Kotlin source code.
 * Used for automated contract testing between SQL queries and mappers.
 */
object FieldExtractor {

    /**
     * Extracts all references to field objects from a specific method in a file.
     * Uses a brace-counting approach to accurately determine method boundaries.
     */
    fun extractFieldsFromMethod(
        relativePath: String,
        methodName: String,
        vararg fieldObjectNames: String
    ): Set<String> {
        val file = java.io.File(relativePath)
        if (!file.exists()) {
            throw IllegalStateException(
                "Source file not found: $relativePath (working dir: ${System.getProperty("user.dir")})"
            )
        }

        val source = file.readText()
        val methodBody = getMethodBody(source, methodName)
            ?: throw IllegalStateException("Method '$methodName' not found in $relativePath")

        return extractFields(methodBody, *fieldObjectNames)
    }

    /**
     * Extracts all references to field objects from Kotlin source code strings.
     * Returns field names with object prefix (e.g., "JobFields.TITLE") to avoid collisions.
     */
    fun extractFields(sourceCode: String, vararg fieldObjectNames: String): Set<String> {
        // Strip comments to avoid extracting fields that are not actually used
        val cleanSource = sourceCode
            .replace(Regex("""//.*"""), "")
            .replace(Regex("""/\*.*?\*/""", RegexOption.DOT_MATCHES_ALL), "")

        val allFields = mutableSetOf<String>()

        // 1. Detect explicit object access: (JobFields|CompanyFields).FIELD_NAME
        if (fieldObjectNames.isNotEmpty()) {
            val objectPattern = fieldObjectNames.joinToString("|")
            val regex = Regex("""($objectPattern)\.([A-Z_]+)""")
            allFields.addAll(regex.findAll(cleanSource).map { 
                "${it.groupValues[1]}.${it.groupValues[2]}"  // Preserve prefix
            })
        }

        // 2. Detect literal string access: get("field_name")
        val literalRegex = Regex("""\.get\(\s*"([A-Za-z_]+)"\s*\)""")
        allFields.addAll(literalRegex.findAll(cleanSource).map { it.groupValues[1] })

        return allFields
    }

    /**
     * Finds the body of a method by counting braces to handle nesting correctly.
     * Internal for testing.
     */
    internal fun getMethodBody(source: String, methodName: String): String? {
        val methodHeaderIndex = source.indexOf("fun $methodName")
        if (methodHeaderIndex == -1) return null

        val firstBrace = source.indexOf("{", methodHeaderIndex)
        if (firstBrace == -1) return null

        var braceCount = 1
        var currentIndex = firstBrace + 1

        while (braceCount > 0 && currentIndex < source.length) {
            when (source[currentIndex]) {
                '{' -> braceCount++
                '}' -> braceCount--
            }
            currentIndex++
        }

        return source.substring(firstBrace, currentIndex)
    }
}
```

**Key Design Decisions:**

| Decision | Rationale |
|----------|-----------|
| **Brace-counting for method boundaries** | Handles nested `if`, `when`, and lambda expressions correctly |
| **Preserve object prefix in output** | `"JobFields.TITLE"` avoids collisions with `"CompanyAliases.TITLE"` |
| **Throw on file/method not found** | Prevents silent test failures from false positives |
| **Support literal string access** | Detects `.get("name")` patterns used in some mappers |
| **Strip comments before parsing** | Avoids extracting fields from commented-out code |

### Step 2: Update JobQueryMapperContractTest (1 hour)

**Key Refinement:** Mappers are refactored into smaller, focused methods (e.g., `mapJobDetailsDto`, `mapJobLocations`, `mapJobCompanyDto`). This enables granular per-method contract validation.

```kotlin
package com.techmarket.contract

import com.techmarket.persistence.job.JobQueries
import com.techmarket.persistence.JobFields
import com.techmarket.persistence.CompanyFields
import com.techmarket.persistence.CompanyAliases
import org.junit.jupiter.api.Test
import org.junit.jupiter.api.Assertions.assertTrue

/**
 * Automated contract tests to ensure JobQueries and JobMapper are in sync.
 */
class JobQueryMapperContractTest {

    private val relativePath = "src/main/kotlin/com/techmarket/persistence/job/JobMapper.kt"

    @Test
    fun `getDetailsSql includes all fields JobMapper read methods require`() {
        val query = JobQueries.getDetailsSql("dataset", "jobs", "companies")

        // Test each sub-method individually
        verifyContract(
            methodName = "mapJobDetailsDto",
            requiredFields = query.requiredFields,
            "JobFields"
        )

        verifyContract(
            methodName = "mapJobLocations",
            requiredFields = query.requiredFields,
            "JobFields"
        )

        verifyContract(
            methodName = "mapJobCompanyDto",
            requiredFields = query.requiredFields,
            "JobFields", "CompanyAliases"
        )
    }

    @Test
    fun `getDetailsSql does not select fields that no JobMapper method reads`() {
        // Union of all fields read by all sub-methods
        val methods = listOf("mapJobDetailsDto", "mapJobLocations", "mapJobCompanyDto", "mapJobRole")
        val allFieldsRead = methods.flatMap { methodName ->
            FieldExtractor.extractFieldsFromMethod(relativePath, methodName, "JobFields", "CompanyAliases")
        }.map { name ->
            lookupFieldValue(name) ?: name
        }.toSet()

        val fieldsQuerySelects = JobQueries.getDetailsSql("dataset", "jobs", "companies").requiredFields.toSet()
        val staleFields = fieldsQuerySelects - allFieldsRead

        if (staleFields.isNotEmpty()) {
            println("⚠️  WARNING: JobQueries.getDetailsSql selects fields no mapper method reads: $staleFields")
        }
    }

    @Test
    fun `getSimilarSql includes all fields JobMapper reads for similar roles`() {
        verifyContract(
            methodName = "mapJobRole",
            requiredFields = JobQueries.getSimilarSql("dataset", "jobs", emptyList()).requiredFields,
            "JobFields"
        )
    }

    private fun verifyContract(
        methodName: String,
        requiredFields: List<String>,
        vararg fieldObjectNames: String
    ) {
        val fieldNames = FieldExtractor.extractFieldsFromMethod(
            relativePath,
            methodName,
            *fieldObjectNames
        )

        val fieldsQueryProvides = requiredFields.toSet()
        val expectedFieldValues = fieldNames.map { name ->
            lookupFieldValue(name) ?: name
        }

        val missingFields = expectedFieldValues.filter { !fieldsQueryProvides.contains(it) }

        assertTrue(
            missingFields.isEmpty(),
            "JobMapper.$methodName reads fields not in requiredFields: $missingFields"
        )
    }

    private fun lookupFieldValue(name: String): String? {
        if (!name.contains(".")) return null  // Literal string access

        val parts = name.split(".")
        val objName = parts[0]
        val fieldName = parts[1]

        val clazz = when (objName) {
            "JobFields" -> JobFields::class.java
            "CompanyFields" -> CompanyFields::class.java
            "CompanyAliases" -> CompanyAliases::class.java
            else -> null
        }

        return try {
            clazz?.getField(fieldName)?.get(null) as? String
        } catch (e: NoSuchFieldException) {
            null
        }
    }
}
```

**Key Changes:**
- ✅ Auto-extracts fields from mapper source with object-aware prefixes
- ✅ Granular method-level testing for complex multi-stage mappers
- ✅ Union-based stale field detection across all sub-methods
- ✅ Standardized `lookupFieldValue` helper for consistent field resolution
- ✅ Clear error messages with exact method names

## Testing Strategy

### Unit Tests for FieldExtractor

**File:** `backend/src/test/kotlin/com/techmarket/contract/FieldExtractorTest.kt`

```kotlin
package com.techmarket.contract

import org.junit.jupiter.api.Test
import org.junit.jupiter.api.Assertions.assertEquals
import org.junit.jupiter.api.Assertions.assertTrue
import org.junit.jupiter.api.Assertions.assertThrows

class FieldExtractorTest {

    @Test
    fun `extractFields finds all JobFields references`() {
        val sourceCode = """
            fun mapJob(r: FieldValueList) {
                val title = r.get(JobFields.TITLE).stringValue
                val source = r.get(JobFields.SOURCE).stringValue
                val country = r.get(JobFields.COUNTRY).stringValue
                val desc = r.get(CompanyAliases.DESCRIPTION).stringValue
            }
        """.trimIndent()

        val fields = FieldExtractor.extractFields(sourceCode, "JobFields", "CompanyAliases")

        assertEquals(
            setOf("JobFields.TITLE", "JobFields.SOURCE", "JobFields.COUNTRY", "CompanyAliases.DESCRIPTION"),
            fields
        )
    }

    @Test
    fun `getMethodBody handles nested braces correctly`() {
        val sourceCode = """
            fun mapJob(r: FieldValueList) {
                if (condition) {
                    val title = r.get(JobFields.TITLE).stringValue
                    when (type) {
                        "A" -> r.get(JobFields.TYPE_A).stringValue
                        "B" -> {
                            r.get(JobFields.TYPE_B).stringValue
                        }
                    }
                }
                val source = r.get(JobFields.SOURCE).stringValue
            }

            fun nextMethod() { ... }
        """.trimIndent()

        val methodBody = FieldExtractor.getMethodBody(sourceCode, "mapJob")
        
        assertTrue(methodBody != null)
        assertTrue(methodBody!!.contains("JobFields.TITLE"))
        assertTrue(methodBody!!.contains("JobFields.SOURCE"))
        assertTrue(methodBody!!.endsWith("}"))
        assertTrue(!methodBody!!.contains("fun nextMethod"))  // Correctly excludes next method
    }

    @Test
    fun `extractFields finds literal string access`() {
        val sourceCode = """
            fun mapLine(row: FieldValueList) {
                val name = row.get("name").stringValue
                val value = row.get("value").longValue
            }
        """.trimIndent()

        val fields = FieldExtractor.extractFields(sourceCode)

        assertEquals(setOf("name", "value"), fields)
    }

    @Test
    fun `extractFields ignores non-matching patterns`() {
        val sourceCode = """
            val jobFields = "something"
            // JobFields.COMMENTED_OUT
            r.get(JobFields.ACTIVE)
        """.trimIndent()

        val fields = FieldExtractor.extractFields(sourceCode, "JobFields")

        assertEquals(setOf("JobFields.ACTIVE"), fields)
    }

    @Test
    fun `extractFieldsFromMethod throws when file not found`() {
        val exception = assertThrows(IllegalStateException::class.java) {
            FieldExtractor.extractFieldsFromMethod(
                "nonexistent/file.kt",
                "someMethod",
                "JobFields"
            )
        }
        assertTrue(exception.message!!.contains("Source file not found", ignoreCase = true))
        assertTrue(exception.message!!.contains("working dir: "))
    }
```

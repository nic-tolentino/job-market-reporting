# Phase 2.1: Salary Normalization Engine

**Status:** Proposed  
**Priority:** HIGH - User Value  
**Effort:** 8-10 hours (updated)  
**Owner:** Backend  

---

## Problem Statement

Salary data from job postings is highly inconsistent:
- **Different currencies:** NZD, AUD, USD, EUR, GBP
- **Different periods:** Hourly, daily, monthly, annually
- **Different formats:** "$120,000", "120k", "$120k-$150k", "€35.000"
- **Different contexts:** Base salary, total compensation, package including super/kiwisaver
- **Gross vs Net:** Some countries (e.g., Spain) discuss salaries in "Bruto" (gross) vs "Neto" (net)
- **"Plus" benefits:** "$120k + super", "$150k + benefits" - need to extract base only

This makes it impossible to compare salaries across job postings or provide meaningful salary insights to users.

---

## Solution Overview

Create a `NormalizedSalary` data structure that captures:
1. **Amount** - Numeric value in the smallest currency unit (e.g., cents)
2. **Currency** - ISO 4217 currency code (NZD, AUD, USD, EUR)
3. **Period** - Time unit (HOUR, DAY, MONTH, YEAR)
4. **Source** - Where the data came from (determines confidence)
5. **isGross** - Whether salary is gross (true) or net (false) - defaults to true

**Confidence Model:**

| Source | Confidence | Description |
|--------|------------|-------------|
| `JOB_POSTING` | HIGH | Explicitly stated in job description text |
| `ATS_API` | HIGH | Structured data from ATS provider API |
| `MARKET_DATA` | MEDIUM | Inferred from market benchmarks |
| `AI_ESTIMATE` | LOW | AI-generated estimate based on role/company |

**Why keep JOB_POSTING and ATS_API separate?**
- **Debugging data quality:** If ATS_API salaries differ systematically from JOB_POSTING, we can detect it
- **Audit trail:** Know exactly which pipeline produced the data
- **Future weighting:** May want to weight ATS data higher for certain analyses

## Data Model

### New Data Class

```kotlin
package com.techmarket.model

/**
 * Represents a normalized salary value with currency, period, and source information.
 * 
 * @property amount The salary amount in the smallest currency unit (e.g., cents)
 * @property currency ISO 4217 currency code (NZD, AUD, USD, EUR)
 * @property period The time period for the salary (HOUR, DAY, MONTH, YEAR)
 * @property source The source of the salary data (determines confidence level)
 * @property isGross Whether this is gross salary (true) or net (false). Defaults to true.
 * @property disclaimer Optional disclaimer for low-confidence estimates
 */
data class NormalizedSalary(
    val amount: Long,
    val currency: String,      // NZD, AUD, USD, EUR
    val period: String,        // HOUR, DAY, MONTH, YEAR
    val source: String,        // JOB_POSTING, ATS_API, MARKET_DATA, AI_ESTIMATE
    val isGross: Boolean = true,
    val disclaimer: String? = null
) {
    companion object {
        // Source constants
        const val SOURCE_JOB_POSTING = "JOB_POSTING"    // HIGH confidence
        const val SOURCE_ATS_API = "ATS_API"            // HIGH confidence
        const val SOURCE_MARKET_DATA = "MARKET_DATA"    // MEDIUM confidence
        const val SOURCE_AI_ESTIMATE = "AI_ESTIMATE"    // LOW confidence
        
        // Period constants
        const val PERIOD_HOUR = "HOUR"
        const val PERIOD_DAY = "DAY"
        const val PERIOD_MONTH = "MONTH"
        const val PERIOD_YEAR = "YEAR"
        
        // Currency constants
        const val CURRENCY_NZD = "NZD"
        const val CURRENCY_AUD = "AUD"
        const val CURRENCY_USD = "USD"
        const val CURRENCY_EUR = "EUR"
        const val CURRENCY_GBP = "GBP"
        
        /**
         * Returns the confidence level based on the source.
         */
        fun getConfidenceForSource(source: String): String = when (source) {
            SOURCE_JOB_POSTING -> "HIGH"
            SOURCE_ATS_API -> "HIGH"
            SOURCE_MARKET_DATA -> "MEDIUM"
            SOURCE_AI_ESTIMATE -> "LOW"
            else -> "UNKNOWN"
        }
        
        /**
         * Country-specific default salary period.
         * Most countries default to annual gross salary.
         */
        fun getDefaultPeriodForCountry(country: String): String = when (country.uppercase()) {
            "NZ", "AU", "US", "UK", "ES", "CA", "IE" -> PERIOD_YEAR
            else -> PERIOD_YEAR  // Safe fallback
        }
        
        /**
         * Country-specific default currency.
         */
        fun getDefaultCurrencyForCountry(country: String): String = when (country.uppercase()) {
            "NZ" -> CURRENCY_NZD
            "AU" -> CURRENCY_AUD
            "US" -> CURRENCY_USD
            "ES", "DE", "FR", "IT", "IE", "NL", "BE" -> CURRENCY_EUR
            "UK" -> "GBP"
            "CA" -> "CAD"
            else -> CURRENCY_NZD  // Safe fallback
        }
    }
    
    /**
     * Converts this salary to an annual amount in the same currency.
     * Uses standard assumptions:
     * - Hourly: 2080 hours/year (40 hrs/week × 52 weeks)
     * - Daily: 260 days/year (5 days/week × 52 weeks)
     * - Monthly: 12 months/year
     * - Yearly: already annual
     */
    fun toAnnualAmount(): Long = when (period) {
        PERIOD_HOUR -> amount * 2080
        PERIOD_DAY -> amount * 260
        PERIOD_MONTH -> amount * 12
        PERIOD_YEAR -> amount
        else -> amount
    }
    
    /**
     * Returns the confidence level for this salary.
     */
    val confidence: String
        get() = getConfidenceForSource(source)
}
```

**Key Parsing Rules:**

| Pattern | Currency | Period | Notes |
|---------|----------|--------|-------|
| `$` + country=AU | AUD | YEAR | Default to annual |
| `$` + country=NZ | NZD | YEAR | Default to annual |
| `€` or `EUR` | EUR | YEAR | European format |
| `/hr`, `/hour`, `p/h` | Detected | HOUR | Explicit hourly |
| `/day`, `per day` | Detected | DAY | Daily rate |
| `/month`, `pcm` | Detected | MONTH | Monthly |
| `/annum`, `p.a.`, `annual` | Detected | YEAR | Explicit annual |
| `k` suffix (e.g., 80k) | Detected | YEAR | Multiply by 1000 |
| European format (35.000€) | EUR | YEAR | Dot = thousands separator (ES) |
| European format (35,000€) | EUR | YEAR | Comma = thousands separator (DE) |
| `$120k + super` | Detected | YEAR | Extract base only, ignore benefits |
| `$100k + benefits` | Detected | YEAR | Extract base only |
| `Salario Bruto 35.000€` | EUR | YEAR | Spanish "Bruto" = gross |
| `Salario Neto 30.000€` | EUR | YEAR | Spanish "Neto" = net (set isGross=false) |

**Parsing Logic Flow:**

```kotlin
fun parseSalary(salaryStr: String?, country: String): NormalizedSalary? {
    if (salaryStr == null) return null
    
    // 1. Detect currency from symbols and country context
    val currency = detectCurrency(salaryStr, country)
    
    // 2. Detect period from keywords (default to country-specific)
    val period = detectPeriod(salaryStr) 
        ?: NormalizedSalary.getDefaultPeriodForCountry(country)
    
    // 3. Check for gross/net indicators (default to gross)
    val isGross = !containsNetIndicator(salaryStr)
    
    // 4. Extract amount (handle European format, stop at '+' for benefits)
    val amount = extractAmount(salaryStr, currency)
    
    // 5. Add disclaimer for low-confidence sources
    val disclaimer = if (source == NormalizedSalary.SOURCE_AI_ESTIMATE) {
        "Estimated based on market data for similar roles"
    } else null
    
    return NormalizedSalary(amount, currency, period, source, isGross, disclaimer)
}
```

**European Number Format Handling:**

```kotlin
private fun extractAmount(salaryStr: String, currency: String): Long {
    // Determine thousands separator based on currency/country
    val thousandsSeparator = when (currency) {
        "EUR" -> "."  // Most EU uses dot for thousands
        else -> ","   // AU/NZ/US use comma
    }
    
    val decimalSeparator = when (currency) {
        "EUR" -> ","  // Most EU uses comma for decimals
        else -> "."   // AU/NZ/US use dot
    }
    
    // Remove currency symbols and whitespace
    var cleaned = salaryStr
        .replace(Regex("[€$£]"), "")
        .replace("EUR", "")
        .replace("USD", "")
        .trim()
    
    // Handle "k" suffix (e.g., 80k → 80000)
    if (cleaned.endsWith("k", ignoreCase = true)) {
        cleaned = cleaned.dropLast(1) + "000"
    }
    
    // Stop at "+" sign (e.g., "$120k + super" → "$120k")
    cleaned = cleaned.substringBefore("+").trim()
    
    // Replace thousands separator with nothing, decimal with dot
    cleaned = cleaned.replace(thousandsSeparator, "")
    
    // Extract digits only (final safety)
    val digits = cleaned.replace(Regex("[^0-9]"), "")
    
    return if (digits.isNotBlank()) {
        digits.toLong() * 100  // Convert to cents
    } else {
        0L
    }
}
```

```typescript
// frontend/src/lib/salary.ts

export interface NormalizedSalary {
    amount: number;       // In cents
    currency: string;     // NZD, AUD, USD, EUR
    period: string;       // HOUR, DAY, MONTH, YEAR
    source: string;       // JOB_POSTING, ATS_API, MARKET_DATA, AI_ESTIMATE
    isGross: boolean;
    disclaimer?: string | null;
}

/**
 * Formats a salary with locale-aware currency display.
 * @param salary The salary object
 * @param locale Optional locale (defaults to en-NZ)
 */
export function formatSalary(salary: NormalizedSalary, locale: string = 'en-NZ'): string {
    const formatter = new Intl.NumberFormat(locale, {
        style: 'currency',
        currency: salary.currency,
        maximumFractionDigits: 0,
    });
    
    const amount = salary.amount / 100;  // Convert from cents
    const period = formatPeriod(salary.period);
    
    return `${formatter.format(amount)}${period}`;
}

/**
 * Formats a salary range.
 */
export function formatSalaryRange(
    min: NormalizedSalary, 
    max: NormalizedSalary,
    locale: string = 'en-NZ'
): string {
    const minFormatted = formatSalary(min, locale);
    const maxFormatted = formatSalary(max, locale);
    return `${minFormatted} - ${maxFormatted}`;
}

/**
 * Returns the confidence badge color based on source.
 */
export function getConfidenceColor(source: string): string {
    switch (source) {
        case 'JOB_POSTING':
        case 'ATS_API':
            return 'text-green-600 bg-green-50';  // HIGH
        case 'MARKET_DATA':
            return 'text-yellow-600 bg-yellow-50';  // MEDIUM
        case 'AI_ESTIMATE':
            return 'text-orange-600 bg-orange-50';  // LOW
        default:
            return 'text-gray-600 bg-gray-50';  // UNKNOWN
    }
}

/**
 * Returns the confidence label.
 */
export function getConfidenceLabel(source: string): string {
    switch (source) {
        case 'JOB_POSTING':
        case 'ATS_API':
            return 'Verified';
        case 'MARKET_DATA':
            return 'Estimated';
        case 'AI_ESTIMATE':
            return 'AI Estimate';
        default:
            return 'Unknown';
    }
}

function formatPeriod(period: string): string {
    switch (period) {
        case 'HOUR': return '/hr';
        case 'DAY': return '/day';
        case 'MONTH': return '/month';
        case 'YEAR': return '';  // Annual is default, no suffix needed
        default: return '';
    }
}
```

## Edge Cases & Mitigations

| Edge Case | Mitigation |
|-----------|------------|
| Salary range without currency (e.g., "80k-100k") | Infer currency from job country |
| "Competitive salary" or "Market rate" | Return null (no structured data) |
| Superannuation included (e.g., "$100k + super") | Parse base only, stop at "+" |
| Package deals (e.g., "$120k including super, car") | Parse base, mark source=MEDIUM |
| Multiple currencies in one posting (rare) | Use primary currency, log warning |
| Ambiguous period (e.g., "$5000" with no context) | Use country-specific default |
| European number format (35.000€) | Use dot as thousands separator for EUR |
| German number format (45.000€) | Use dot as thousands separator for EUR |
| Spanish "Bruto" vs "Neto" | Detect keywords, set isGross flag |
| "$120k + 11% super" | Extract base only (120k), ignore super % |
| Salary in crypto/equity only | Return null, note in benefits field |
| Daily rate without explicit period | Default to DAY if contains "day"/"daily" |
